# Kimi -> GPT

## 本轮前端改动总结（深夜模式主题切换）

### 1. 主题切换按钮位置

**设置页面 → 通用标签 → 界面语言下方**

新增了一个主题切换按钮组，包含"浅色"（太阳图标）和"深夜"（月亮图标）两个选项，当前选中的主题有激活高亮样式。

### 2. 用户可切换的主题

- **浅色（light）**：保持原有的 Sanaka 浅色主题
- **深夜（dark）**：暗紫/深夜/柔和方向，背景为深紫灰（#1a1625），不是纯黑

### 3. 改动的前端文件

| 文件 | 改动内容 |
|------|----------|
| `src/domain/schemas.ts` | `appSettingsSchema` 新增 `theme: z.enum(['light', 'dark']).default('light')` |
| `src/domain/defaults.ts` | `defaultSettings` 新增 `theme: 'light'` |
| `src/store/AppStore.tsx` | 新增 `setTheme` 方法，应用启动时同步 `data-theme` 到 DOM，切换时更新 settings |
| `src/pages/SettingsPage.tsx` | 新增主题切换 UI（SunIcon/MoonIcon + theme-toggle 按钮组） |
| `src/styles/app.css` | 新增 `:root[data-theme="dark"]` 完整变量集 + `.theme-toggle` 样式 |
| `src/i18n/resources.ts` | 新增中英文翻译：`theme`、`light`、`dark` |

### 4. 颜色体系收干净情况

- **已统一**：所有主要 CSS 变量（bg、panel、text、primary、console 等）都已在 dark 主题下重新定义
- **控制台**：深夜模式下控制台顶栏、抽屉、状态提示均使用主题变量自动切换
- **已知遗留**：CSS 中仍有部分硬编码的 rgba 值（如阴影、边框透明度、渐变等），这些在深浅色模式下视觉差异不大，暂未全部替换。核心背景/文字/面板色已完全变量化。

### 5. 技术实现要点

- 主题值保存在 AppSettings 中，持久化到磁盘
- 通过 `document.documentElement.setAttribute('data-theme', theme)` 切换
- CSS 使用 `:root[data-theme="dark"]` 覆盖变量值
- 切换即时生效，无需重启
