# Kimi -> GPT：共享剪贴板 / Sanaka Tools ISO 前端方案

## 1. 主入口位置

**推荐：控制台顶部工具栏右侧，紧挨着「更多」按钮。**

理由：
- 共享剪贴板是「运行态」功能，只在虚拟机运行时有意义
- 控制台顶部工具栏已经集中了所有运行态操作（换盘、重启、关机）
- 用户不需要离开当前上下文就能查看/操作
- 与「共享文件夹」保持同一层级认知（都是运行态集成能力）

图标建议：一个「剪贴板 + 双向箭头」的简洁图标，18x18，线框风格与现有工具栏一致。

## 2. 面板形式

**推荐：右侧滑出抽屉（与 Info Drawer 同方向、同动画）。**

理由：
- 与现有 Info Drawer 保持一致的交互模型，降低学习成本
- 右侧抽屉不遮挡 VNC 画面主体
- 空间足够放下「说明区 + 操作区 + 状态区」
- 动画可直接复用 `.console-drawer` 的 240ms ease-out

面板标题：**「共享剪贴板」**

## 3. 面板信息结构（从上到下）

## 4. 状态设计

### 状态 1：已关闭（默认）
- 开关：关闭
- 状态区：灰色圆点 + 「已关闭」
- 说明文字：「开启后，主机与客户机可以共享剪贴板内容」
- 操作按钮：禁用

### 状态 2：已启用，等待客户机工具连接
- 开关：开启
- 状态区：黄色/橙色圆点 + 「等待客户机…」
- 说明文字：「已启用。请在客户机中安装 Sanaka Tools 以建立连接」
- 操作按钮：「挂载 Sanaka Tools」可用

### 状态 3：已连接
- 开关：开启
- 状态区：绿色圆点 + 「已连接」
- 说明文字：「主机与客户机可以双向复制粘贴文本」
- 操作按钮：「重新挂载 Sanaka Tools」（如果需要重装）

### 状态 4：连接断开 / 重连中
- 开关：开启
- 状态区：黄色/橙色圆点 + 「正在重新连接…」
- 说明文字：「连接暂时中断，正在尝试恢复」
- 操作按钮：可用

### 状态 5：错误态
- 开关：开启
- 状态区：红色圆点 + 「连接出错」
- 说明文字：具体错误信息（如「客户机工具版本不兼容」）
- 操作按钮：「重新挂载 Sanaka Tools」

## 6. 视觉方向

- **状态圆点**：复用现有 `console-topbar__dot` 体系（绿/黄/红）
- **说明区**：使用 `.info-panel` 或类似的浅色卡片，圆角 12px，内边距 16px
- **步骤列表**：使用有序列表，数字用圆形徽章（与模板列表风格一致）
- **开关**：复用现有的 `.ios-toggle` 组件
- **分隔线**：使用 `var(--line)` 颜色的 1px 实线
- **整体风格**：与 Info Drawer 保持一致，不引入新视觉语言

## 7. 与共享文件夹的关系

建议共享剪贴板和共享文件夹在控制台的入口并列放置：
- 顶部工具栏右侧：「共享文件夹」图标 → 左滑抽屉
- 顶部工具栏右侧：「共享剪贴板」图标 → 右滑抽屉

或者如果空间有限，两者共用一个「集成」图标，点击后展开下拉菜单选择「共享文件夹 / 共享剪贴板」。
