## 当前问题

我这边后端已经开始做了：

- 每台运行中的虚拟机独立起一个 TCP 剪贴板桥
- 运行态会有 `clipboardBridge` 状态
- 机器配置会有 `integration.clipboard`

但前端还没有一套清晰的用户界面定义。

现在最需要的是：

- 这个功能在界面里应该长什么样
- 用户如何理解“要装工具盘”
- 用户如何理解“已连接 / 未连接 / 等待客户机工具”

## 你需要产出什么

只产出设计稿示例，不写任何正式代码。

请给我一份完整的前端方案，至少包含：

1. 入口放在哪里

- 是在控制台右侧状态栏
- 还是在详情页
- 还是两边都要有，但主入口明确

2. 面板信息结构

至少考虑这些内容怎么排：

- 共享剪贴板开关
- 当前状态
- 当前仅支持文本剪贴板
- 挂载 `Sanaka Tools ISO`
- 客户机安装说明
- 安装后重启提示

3. 状态设计

至少给出这些状态怎么表现：

- 已关闭
- 已启用，等待客户机工具连接
- 已连接
- 连接断开 / 重连中
- 错误态

4. 文案方向

文案要像 Sanaka，别写得太工程味。

尤其是：

- “共享剪贴板”
- “等待客户机工具连接”
- “请在客户机中运行 Sanaka Tools 安装程序”
- “安装完成后请重启客户机”
- “当前仅支持文本剪贴板”

5. XP 场景下的说明区怎么做

因为这个功能第一版明确是 XP 优先，所以你要考虑：

- 说明区不要太现代花哨
- 但也别太土
- 重点是让用户看懂下一步做什么
