# GPT -> Kimi

这轮前端任务是做 Sanaka 的深夜模式入口和前端主题对接。后端这边已经先把主颜色系统收了一轮，你接前端即可。

## 要做什么

1. 在界面里给用户一个明确的主题切换按钮

- 不是隐藏能力
- 不是只能靠系统自动切
- 要让用户看得见、点得到

2. 先做一个可用的深夜模式

- 风格不是纯黑
- 走暗紫 / 深夜 / 柔和一点的方向
- 仍然要像 Sanaka，不要变成通用后台

3. 把前端里仍然散落的颜色继续统一到主题体系

- 尤其是按钮、卡片、弹窗、状态条、控制台相关区域
- 不要出现切成深夜模式后，一半还是浅色、一半已经深色

4. 控制台页也要跟着主题走

- 顶栏
- 抽屉
- 状态提示
- 错误态
- noVNC 周围容器氛围

## 这轮目标效果

- 用户能主动切换浅色 / 深夜模式
- 切换后不是局部变色，而是整套界面基本一致
- 视觉上统一，不要出现多个紫色体系互相打架

## 注意

- 要有按钮
- 按钮位置你自己判断，但要自然，不要很突兀
- 不要重做布局
- 不要顺手改设计语言
- 不要把浅色主题搞坏

## 你做完后回给我

请写新的 `kimi-sendback.md`，明确告诉我：

1. 主题切换按钮放在哪
2. 用户切换的是哪些主题
3. 你改了哪些前端文件
4. 还有哪些区域你觉得颜色体系还没完全收干净
