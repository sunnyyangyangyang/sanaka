# 感想

这张图里，Sanaka 的控制台界面已经不是“能打开一个虚拟机窗口”那么简单了，而是把一个完整可用的 Windows XP 办公桌面稳定地放进了自己的显示层里。

画面顶部是 Sanaka 的深色控制台状态栏。左侧有返回箭头和机器名 `XP`，中间是绿色状态点和 `运行中`，右侧是缩放模式与一排控制按钮。整个顶栏非常克制，没有抢虚拟机画面的存在感，但又明确告诉人：这不是截图软件，不是贴图，这是一台正在运行的虚拟机。

主体画面里，Windows XP 桌面背景是经典的蓝天绿地。前景打开的是 Microsoft Excel，标题栏写着 `Microsoft Excel - Book1`。界面语言是中文，菜单栏是：

- 文件
- 编辑
- 视图
- 插入
- 格式
- 工具
- 数据
- 窗口
- 帮助

工具栏、字体栏、格式栏都完整显示，说明这不是一个残缺环境，而是一个真正能工作的旧版 Office 界面。

表格内容也非常具体，不是空白演示页。第一行表头依次是：

- 工号
- 姓名
- 部门
- 入职日期
- 工资
- 工资等级

下面几行已经录入了员工数据，例如：

- `1001 张华 销售部 2019-12-28 4800 低`
- `1002 刘芳 销售部 2020-4-7 5200 中`
- `1003 李明 技术部 2020-8-16 7600 中`
- `1004 呆呆 研究部 2007-6-25 23280 高`
- `1005 白姿 研究部 2008-4-25 21682 高`

其中第 4 行和第 5 行整行被粉色高亮，当前选中单元格在 `G8`，黑色边框清晰可见。底部还能看到：

- `Sheet1`
- `Sheet2`
- `Sheet3`

右下角系统托盘、开始菜单按钮、任务栏里的 Excel 标签也都在，说明整套旧系统桌面逻辑是完整活着的，而不是只有一个应用窗口。

我对这张图的感想很直接：它说明 Sanaka 已经跨过“虚拟机前端原型”这道坎了。现在看到的不是参数、不是日志、不是 QEMU 命令行，而是用户会真正拿来工作的结果。一个 XP 虚拟机里打开了中文 Excel，表格能看、能选、能像一台老电脑那样摆在眼前，这件事本身就已经很有说服力。

更重要的是，这张图带来的不是抽象成就感，而是一种具体的完成感。你能从画面里一下子判断出很多东西已经打通了：

- QEMU 启动稳定
- Sanaka 控制台显示稳定
- noVNC 路线可用
- 输入与分辨率至少已经到了可操作程度
- 老系统和老软件场景不是纸上谈兵

而且这不是随便选了一个桌面截图。XP 加 Excel，本身就带着一种非常强的“旧软件兼容性验证”意味。它证明 Sanaka 不只是能跑新系统，也能碰那些最容易暴露问题的旧环境：老显卡、老输入逻辑、老分辨率、老 UI、老软件。

如果说之前很多节点是“后端架构搭起来了”“QMP 接进来了”“VNC 通了”“换盘能用了”，那这张图就是把这些离散胜利收束成一个用户能瞬间理解的结论：

Sanaka 已经能把一台真正可用的老 Windows 机器放到你面前，而且看起来像一件完整产品，而不是实验脚手架。
