# Sanaka 构建方法

这份文档只写现在仓库里实际能用的构建方式，尽量说人话。

## 先说结论

- macOS：现在主流程是打出 `.app` 目录，并把本地编译好的 QEMU 塞进去
- Windows：可以打安装包，也可以先打 `win-unpacked` 再嵌入本机的 QEMU
- Linux：当前仓库能直接打 `AppImage`

## 通用准备

先进入项目目录：

```bash
cd /你的路径/sanaka
```

安装依赖：

```bash
npm install
```

如果只是想先检查前端和主进程代码是否正常：

```bash
npm run build
```

## macOS 构建

### 你需要先准备什么

- 一台 macOS 机器
- 已经能正常跑 `npm install`
- 一份你自己编译好的 QEMU 构建目录

现在仓库里的 macOS 打包脚本是：

```bash
npm run pack:mac
```

它实际等价于：

```bash
bash ./scripts/package-sanaka-macos.sh /Volumes/sks/src/qemu-11.0.1/build-sanaka
```

也就是说，默认写死使用这个 QEMU 构建目录：

```bash
/Volumes/sks/src/qemu-11.0.1/build-sanaka
```

如果你的 QEMU 不在这里，就直接手动传路径：

```bash
bash ./scripts/package-sanaka-macos.sh /你的/qemu/build目录
```

### 构建结果在哪

通常会在这里出现：

```bash
release/mac-arm64/Sanaka.app
```

脚本会做这些事：

- `npm run build`
- `electron-builder --dir --mac`
- 把 QEMU 嵌进 `Sanaka.app`
- 刷新 macOS 的应用注册

## Windows 构建

### 方式 1：直接打安装包

这个命令会直接走 `electron-builder` 的 NSIS 安装包流程：

```bash
npm run pack:win
```

产物一般在：

```bash
release/
```

通常会得到安装程序 `.exe`。

### 方式 2：先打目录版，再嵌入 QEMU

如果你想先做一个可运行目录，再把 `C:\\Program Files\\qemu` 里的 QEMU 塞进去，用这个：

```bash
npm run pack:win:dir
```

它实际调用：

```bash
bash ./scripts/package-sanaka-windows.sh
```

这个脚本会自动找这些目录里的 QEMU：

```text
C:\Program Files\qemu
C:\Program Files\QEMU
```

如果自动找不到，就手动传路径：

```bash
bash ./scripts/package-sanaka-windows.sh "C:/Program Files/qemu"
```

构建结果通常在：

```bash
release/win-unpacked
```

### Windows 上推荐怎么跑

如果你是在 Windows 上本地打包，推荐顺序是：

```bash
sh scripts/pull-main.sh
npm install
npm run pack:win
```

如果只是想要目录版：

```bash
sh scripts/pull-main.sh
npm install
npm run pack:win:dir
```

## Linux 构建

Linux 现在仓库里直接用：

```bash
npm run pack:linux
```

它会调用：

```bash
npx electron-builder --linux
```

当前配置目标是：

```text
AppImage
```

产物一般在：

```bash
release/
```

## 常见问题

### 1. `npm install` 失败怎么办

先确认：

- Node.js 版本别太怪
- 网络能正常下载 Electron
- 代理已经配好

如果是 Windows 下载 Electron 失败，通常先配镜像再装。

### 2. macOS 为什么不能只靠 `npm run pack:mac`

因为现在 Sanaka 的 macOS 包不是单纯 Electron 壳子，还要把 QEMU 塞进去，所以需要脚本额外处理。

### 3. Windows 的 NSIS 是不是会输出安装程序

是。`npm run pack:win` 的目的就是生成安装程序。

### 4. Linux 现在会不会自动内置 QEMU

当前这份仓库流程里，没有专门给 Linux 做和 macOS/Windows 一样的 QEMU 内嵌脚本。

## 现在最常用的命令

### macOS


### Windows 安装包

```bash

```

### Windows 目录版

```bash
bash ./scripts/package-sanaka-windows.sh "C:/Program Files/qemu"
```

### Linux

